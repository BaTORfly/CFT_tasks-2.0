## Практическая задача 2 - Фигуры
* Написать консольное приложение с объектно-ориентированной архитектурой, выводящее <br>
характеристики заданной геометрической фигуры. <br>

### Входные данные
Тип фигуры и её параметры читаются из файла. Для разных типов фигур параметры отличаются. <br>

Первой строкой в файле идёт код типа фигуры. Второй строкой - параметры фигуры через пробел. <br>

#### Пример:
```text
CIRCLE
5
```

### Выходные данные
Результатом работы приложения являются вычисленные характеристики фигуры. Для разных <br>
фигур характеристики отличаются, так же они зависят от введенных параметров фигуры.
Результат выводится в файл или консоль, это конфигурируется через аргумент командной строки.

#### Пример: 
```text
Shape type: CIRCLE
Area: 78.53 sq. mm
Perimeter: 
Radius: 5 mm
Diameter: 10 mm
```

#### Требования к работе приложения

  Единицы измерения параметров фигуры не имеют значения, главное, чтобы они <br>
соответствовали друг другу и введенным данным и были указаны в выходной информации. <br>

  Приложение должно поддерживать типы фигур и характеристики для них из таблицы ниже. <br>

  Общие характеристики для всех типов фигур:
  * Название
  * Площадь
  * Периметр

| Название      | Тип (код) | Параметры        | Выходная информация                                                           |
|---------------|-----------|------------------|-------------------------------------------------------------------------------|
| Круг          | CIRCLE    | Радиус           | - Радиус<br>- Диаметр                                                         |
| Прямоугольник | RECTANGLE | Длины сторон (2) | - Длина диагонали<br>- Длина (длинная сторона)<br>- Ширина (короткая сторона) |
| Треугольник   | TRIANGLE  | Длины сторон (3) | Для каждой из трех сторон:<br>- Длина стороны<br>- Противолежащий угол        |

В приложении должно использоваться логирование (вместо вывода на консоль или System.err), <br>
исключение - вывод на консоль результата программы, если того требует аргумент командной строки. <br>
Должен быть настроен логгер, выводящий логи в консоль, логгер, выводящий логи в файл - по желанию. <br>

* Код должен быть покрыт unit-тестами.
